# 文学ディストロ 研究手法記録

100作品×6ディストロ v2クリーンデータベースの設計・構築方法のドキュメント。
「どう作ったか・なぜこう設計したか」を記録する。用途・問いは別途定義する。

---

## 0. 研究の基本フレーム

### 文学テキストをログとして読む

この研究の基本的な問いは：**人間のOSに相当する構造を、文学テキストから露出させられるか**。

Spiral Recursive OS は日次ログの逐次投入とマージによって、個人のOS的構造（判断パターン・反復・回復プロセス等）を浮かび上がらせる。
同じ原理を、100文化圏の文学テキストに適用する。

| Spiral OS | 文学ディストロ |
|-----------|--------------|
| 日次ログ（個人の雑メモ） | 文学テキスト（文化圏のログ） |
| 6ディストロ読解 | 6ディストロ並列分析 |
| fragments への蒸留 | `_result.md`（チャンク別） |
| knowledge compiler（Sonnet） | `merge_chunks.py`（Sonnet） |
| wiki / core 収束命題 | `db/observations.json` |

文学テキストは「その文化圏が重要と判断した経験の記録」として扱う。
6ディストロは、そのテキストに異なる認識枠を当て、**単一フレームでは見えない構造を取り出す**ための観測装置として機能する。

### 収束命題の意味

3ディストロ以上で独立に同じ構造が出現した場合を「収束命題」と呼ぶ。
収束命題は「ある文化圏のOSに組み込まれている構造」の候補として扱う。

100文化圏を横断して収束命題が共有されているとき、それは「人間のOSの候補命題」となる。

## 1. 6ディストロ並列分析の設計

### ディストロとは

1つのテキストを6つの異なる認識枠（ディストロ）で独立に読む。
各ディストロは互いに参照せず、同じテキストを別の角度から観測する。

### 6ディストロの構成と読解角度

| ディストロ | 読解角度 |
|-----------|---------|
| **EK（経験知）** | 主体の行動・判断・生存戦略のパターン。「この局面でどう動くか」の実践知 |
| **医療** | 心身の反応・ストレス・喪失・回復・慢性適応のプロセス |
| **社会学** | 制度・権力・役割・集団・排除のメカニズム |
| **精神分析** | 欲動・防衛・反復・対象関係・同一性の構造 |
| **文学分析** | 語り構造・形式・反復・視点・テキストの編集原理 |
| **宗教** | 超越・禁忌・儀礼・罰・救済・秩序回復の論理 |

### 選定の理由

- 互いに還元不可能な6つの認識論を選ぶことで、単一フレームでは見えない構造を取り出す
- 3ディストロ以上で独立に同じ構造が出現した場合、「収束命題」として扱う
- 6分の1しか出現しない観測は「固有観測」として記録する（排除しない）

### 各ディストロの出力構造

```
### core
**C-n**: タイトル
定式: 「一般化された命題文」

### theory
T-n: このテキスト固有の理論的観察

### tips
Tip-n: 読解上の注意点・切り口
```

`core` が定量分析の主対象。`theory` と `tips` はテキスト固有の補助情報。

## 2. パイプライン

```
logs/{region}/{work_key}.txt
    ↓ Codex v2（run_codex_v2.sh）
fragments/{region}/{work_key}_result.md
    ↓ Anthropic Sonnet（run_knowledge_compiler.py）
knowledge/literature/{work_key}.md
    ↓ パーサー（build_db.py）
db/works.json
db/observations.json
```

### 各層の役割

**logs/** — 生テキスト（60,000字上限）  
テキスト入手先: Gutenberg / 青空文庫 / ctext.org / OpenITI / Internet Archive

**fragments/** — 6ディストロ並列分析結果  
Codex が `codex_analysis_prompt.md` のテンプレートに従って生成。  
frontmatter に `pipeline_version: v2` を含む。

**knowledge/literature/** — 収束観測コンパイル  
Anthropic Sonnet が fragments を読んで収束命題・固有観測・OS的含意を生成。  
`run_knowledge_compiler.py` で自動生成。

**db/** — フラット化した定量分析用データ  
`build_db.py` が全 fragments をパースして生成。再生成可能な派生物。

## 3. v2クリーン化の判断

### Kafka contamination（カフカコンタミ）とは

v1パイプラインでカフカ「変身」を最初に分析した際、その結果（苦悩・相転移・カタルシスのフレーム）が後続作品の分析に混入した現象。

具体的には：
- ナレッジテンプレートがカフカ分析の語彙で設計された
- 「苦悩スコア」「カタルシス類型」「相転移」が分析の前提として埋め込まれた
- 結果として、苦悩を明示していない作品にも苦悩フレームが当てはまってしまった

### v2での対処

**判断ルール（`run_knowledge_compiler.py` SYSTEM プロンプトより）**

- 収束命題はテキストから自然に出てきた構造を記述する
- 苦悩・カタルシス・不条理などのフレームを前提にしない
- 苦悩が出てきていない作品で苦悩フレームを当てはめない
- 他の作品との比較セクションは追加しない
- `connections` に `happiness_resonance.md` を自動追加しない

**削除した v1 成果物**

- `fragments/_v1_backup/`（176ファイル）
- `fragments/_archive/`（9ファイル）
- `knowledge/literature/`（旧88ファイル）
- `meta/diffs/`（91ファイル）
- `core/` `theory/` `playbooks/` `tips/` ディレクトリ

いずれも git 履歴に保存されている。

## 5. 限界と注意事項

### テキストの性質

- **60,000字保存・30,000字読解**: `logs/` には60k字保存しているが、Codex CLIの内部Read上限により実際に分析されているのは約30k字。長編は冒頭4〜5%程度しか読めていない可能性がある（2026-04-08判明）
- **英訳テキスト**: 非英語作品の多くは英訳版。翻訳者の解釈が混入する
- **OCRノイズ**: Internet Archive の djvu.txt は誤認識が多い。意味が取れれば許容している

### 口承文学の特性

- 著者ではなく「語り手・文化的集合知」として処理している
- 民族誌的記録者（Rattray, Parker 等）の解釈が間接的に混入する可能性がある
- 宗教・精神分析ディストロは口承文化と親和性が高い。EKディストロは「この文化での経験則」として読み替えている

### PD制限

- 1928年以前の作品を原則とする
- W.R.Smith「Australian Myths」1930年版は IA 認証制限により取得不可 → Parker 1897年版（PG #3833）で代替
- OpenITI は CC BY-NC-SA（非商用限定）

### ディストロ適用の限界

- 6ディストロは西洋近代の認識枠を基盤としている
- 集合的主体（ネイティブアメリカン）・関係的自己（オセアニア）では個人主体前提のディストロが機能しない場合がある
- その場合の「空欄」や「ずれ」自体がデータとして有効
