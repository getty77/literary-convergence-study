# 文学ディストロ v3 研究フレーム

## v2との関係

v2（N=100、6ディストロ並列分析）は以下の理由で破棄し、v3として再分析する。

| 問題 | 内容 |
|------|------|
| 技術的問題 | Codex CLI の内部Read上限により長編は冒頭30,000字しか読めていなかった |
| フレーム問題 | 「作品のOS骨格を露出させる」視点で読んでいなかった |

手法が混在した状態での分析継続は研究として不誠実なため、v2を破棄してv3を新実験として立て直す。  
v2データは `meta/_v2_archive/` に移す（git履歴にも保存されている）。

## v3の基本フレーム

### 問い

**文学テキストを「ログ」として読み、そのテキストを動かしているOSの骨格を露出させられるか。**

Spiral Recursive OS は日次ログの逐次投入とマージによって個人のOS的構造を浮かび上がらせる。  
v3 では同じ原理を文学テキストに適用する。

| Spiral OS | 文学ディストロ v3 |
|-----------|----------------|
| 日次ログ（個人の雑メモ） | 文学テキスト（文化圏のログ） |
| 逐次投入→マージ | チャンク分割読み→断片マージ |
| OS的構造の露出 | 作品が動いているOSの骨格露出 |

### OS骨格とは

作品の表層（ストーリー・登場人物）の背後にある、  
その作品を「動かしている」判断・反復・回復・秩序の論理構造。

これは「作者の思想」でも「文化的背景の解説」でもなく、  
**テキスト自体の振る舞いから浮かび上がる構造**として扱う。

## v3パイプライン設計方針

### 全文カバレッジ

v2の30,000字制限を解消する。  
長編はチャンク（単位未定）に分割し、チャンクごとにOS断片を抽出、最後にマージする。

```
テキスト全文
    ↓ チャンク分割
chunk_01, chunk_02, ... chunk_N
    ↓ 各チャンクからOS断片を抽出
os_fragment_01, os_fragment_02, ... os_fragment_N
    ↓ マージ（収束・矛盾・固有を整理）
OS骨格露出レポート
```

この構造は Spiral OS の「逐次ログ投入→マージ」と同型であり、  
OS自身の動作原理を実証する形でもある。

### チャンク設計（検討中）

- チャンクサイズ：未定（作品の章構成に合わせる案あり）
- 前のチャンクのOS断片を次チャンクの入力に含める（文脈引き継ぎ）
- 各チャンクへのプライミング：螺旋OS語彙（ディストロ、層など）を毎回付与

### 実行エージェント（検討中）

| 案 | 特徴 |
|----|------|
| Claude Code（Sonnet） | 螺旋OS語彙理解済み、分析精度が高い |
| Codex CLI | チャンク×マージ構造と相性良い、コスト検討が必要 |

チャンク分割→断片出力→マージというフローであれば、どちらでも構造は成立する。  
まず小規模実験で比較する。
