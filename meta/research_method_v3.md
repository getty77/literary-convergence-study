# 文学ディストロ v3 実験手法定義

---

## 0. 研究の問い

**文学テキストから、その文化圏のOSに相当する構造を露出させられるか。**

Spiral Recursive OS は日次ログの逐次投入とマージによって個人のOS的構造を浮かび上がらせる。
同じ原理を、複数の文化圏の文学テキストに適用する。

文学テキスト = その文化圏がOSに相当するものを記録したログ  
EK分析 = ログからOS骨格（行動・判断・生存戦略のパターン）を抽出する観測装置

---

## 1. ディストロ設計

### 6ディストロ並列

**理由：**
- 収束命題の測定にはディストロ間の独立観測が必要（3ディストロ以上で同じ構造 = OS候補命題）
- N=10程度のパイロットなら6ディストロのコストは許容範囲
- 行動・身体・社会・心理・語り・宗教の多層でOS構造を記述できる

### 6ディストロの読解角度とOS骨格の問い

| ディストロ | 読解角度 | OS骨格として問うこと |
|-----------|---------|-------------------|
| **EK（経験知）** | 主体の行動・判断・生存戦略 | このOSはどんな状況判断ルールと生存戦略を持っているか |
| **医療** | 心身の反応・喪失・回復・慢性適応 | このOSはどのような身心応答・回復機構を持っているか |
| **社会学** | 制度・権力・役割・集団・排除 | このOSはどのような社会メカニズムを内蔵しているか |
| **精神分析** | 欲動・防衛・反復・対象関係 | このOSの心理構造・反復パターンはどのような骨格か |
| **文学分析** | 語り構造・視点・反復・編集原理 | このOSはどのような形式でみずからを記述しているか |
| **宗教** | 超越・禁忌・儀礼・罰・救済 | このOSはどのような超越応答・秩序回復の論理を持つか |

### 収束命題の定義

3ディストロ以上で独立に同じ構造が出現した命題を「収束命題」として扱う。
収束命題はそのOSのより深い骨格候補として優先的に扱う。

### 出力構造

```
### core
**C-n**: タイトル
定式: 「〜のとき、〜が起きる」または「〜というOS構造を持つ」

### theory
T-n: テキスト固有の補助観察

### tips
Tip-n: 読解上の注意点
```

## 2. コーパス選定基準

### 基本条件（全て満たすこと）

1. **文化的正典性**: その文化圏が「重要」と判断してきたテキスト。個人の好みでなく、文化的蓄積・継承・引用の実績で判断する。
2. **公開テキスト取得可能**: 著作権切れのPDテキストが実際に取得できること。LLM学習データ知識からの分析は不可。
3. **全文読解可能**: チャンク分割して全文を機械的に処理できること。

### パイロット対象（N≈10）

広域の文化圏をカバーし、各圏の価値基準に食い込んでいるテキストを優先する。
一文化圏に偏らないよう設計する。

### 言語方針

原語・英訳の混在を許容する。
日本語テキスト（青空文庫）、中国語テキスト（ctext.org）、アラビア語テキスト（OpenITI）、英訳テキスト（Gutenberg等）はいずれも使用可。

翻訳者の解釈混入は分析上の制限として `research_notes` に記録する。

## 3. パイプライン

```
全文テキスト取得
    ↓
logs/{region}/{work_key}.txt（全文、文字数上限なし）
    ↓ chunk_splitter.py（チャンクサイズ: 200k字、オーバーラップ: 20k字）
logs/{region}/.chunks/{work_key}/chunk_{n:02d}.txt
    ↓ codex（codex_analysis_prompt.md v3）
fragments/{region}/.chunks/{work_key}/chunk_{n}_result.md
    ↓ merge_chunks.py（Anthropic Sonnet）
fragments/{region}/{work_key}_result.md
    ↓ run_knowledge_compiler.py（Anthropic Sonnet）
knowledge/literature/{work_key}.md
    ↓ build_db.py
db/works.json / db/observations.json
```

### 各ステップの役割

**chunk_splitter.py**: 全文を200k字チャンクに分割（段落境界優先）

**codex（v3プロンプト）**: EKディストロでOS骨格を抽出。全チャンクを均一手法で処理。

**merge_chunks.py**: チャンク別EK分析をSonnetで統合・重複除去。OSの骨格命題として整理。

**run_knowledge_compiler.py**: 収束命題・固有観測・OS的含意をコンパイル。

**build_db.py**: 全作品のC-nエントリをフラットなJSONとして出力。横断分析用。

---

## 4. 限界と注意事項

- **翻訳テキスト**: 非英語・非原語作品の英訳は翻訳者の解釈が混入する
- **EK単一の限界**: 身体・制度・心理・宗教の次元はカバーしない。OSの行動実行レイヤーに限定した観測になる
- **チャンク境界**: チャンク内で完結しない構造変化は見逃す可能性がある
- **文化正典の偏り**: 現存する「正典」は多くが権力・制度による選別を経ている
